# Технічні деталі під повідомленням ШІ

**Дата:** 2026-06-20
**Статус:** затверджено до планування

## Мета

Під відповіддю ШІ (у тому самому рядку, де зараз показано відносний час
надсилання) додати іконку **(i)**. Клік відкриває поповер із технічними
деталями відповіді: модель, токени, час роботи ШІ, точна дата/час, і — лише
для адмінів — вартість у $.

Іконка показується **тільки на повідомленнях асистента**. Повідомлення
користувача лишаються без змін.

## Контекст і обмеження

- Реальні користувачі — нетехнічний персонал компанії на спільному
  адмінському ключі. Тому вартість у $ чутлива й показується лише адмінам
  (`isAdmin` уже прокидається в `ChatMessage`). Токени/час/модель бачать усі.
- Токени та вартість **уже рахуються** й пишуться в таблицю `usage`
  (`runner.ts` → `recordUsage`), прив'язані до `messageId`. Але вони не
  приєднуються до повідомлень при читанні історії і не потрапляють у
  `metadata`, яку бачить UI.
- Час роботи (`elapsedMs = Date.now() - startedAt`, `runner.ts:703`)
  обчислюється, але лише відправляється в realtime-подію — **не персиститься**,
  тож після перезавантаження сторінки зникає.

## Рішення

### 1. Персистування метаданих (бекенд)

У фінальному `db.update(messages)` в `runner.ts` (де вже формується
`metadata: { taskId, status, parts, ... }`, ~рядок 691) додати, лише для
успішного завершення (`finalStatus === "completed"`), коли є дані:

- `durationMs: number` — те саме значення `Date.now() - startedAt`.
- `usage: { input: number; output: number; cached: number }` — ті самі числа,
  що йдуть у `recordUsage` (non-cached input, output, cacheRead).
- `costUsd: number | null` — обчислена вартість (`costUsd(modelId, usage)`).
- `model: string` — `modelId`.

Запис у таблицю `usage` лишається без змін — він живить аналітику
`/settings/usage`. Метадані повідомлення — денормалізована копія, щоб UI не
робив JOIN (той самий патерн, що з `parts`).

Примітка щодо порядку: зараз `recordUsage` викликається **після**
`db.update(messages)`. Оскільки тепер ті самі числа (токени + вартість)
потрібні і в metadata, обчислення `usage`/`cost` треба підняти **до**
`db.update(messages)`, а сам запис у таблицю `usage` лишити де він є (або
поряд). Обчислення вартості (`costUsd`) не повинно валити фіналізацію —
обгорнути в try/catch, при помилці metadata просто не отримує `costUsd`.

### 2. Типи й презентер

- `contracts.ts` → `MessageMeta` отримує опційні поля:
  `durationMs?`, `usage?: { input; output; cached }`, `costUsd?`, `model?`.
- `presenter.ts` → `toUIMessages` прокидає ці поля у UI-metadata-об'єкт
  (поряд з `createdAt`, `taskStatus` тощо).

### 3. UI (`message.tsx`)

- `ChatMessageImpl` читає нові поля з `metadata` і передає їх у рядок дій
  асистента (той блок, що рендерить `TimestampRow` для асистента, ~рядок 806).
- Новий компонент **`MessageDetails`** (кнопка-(i) + поповер). Рендериться
  лише якщо є хоч одне поле деталей (`durationMs`/`usage`/`model`). Кнопка
  стоїть поряд з `TimestampRow`, у тому самому ряду дій, що й Copy/Regenerate.
- Вміст поповера (локалізований), рядок-у-рядок:
  - **Модель** — людська назва (з `model`; якщо є каталог — `displayName`,
    інакше сам id).
  - **Токени** — `вхід · вихід` (+ `кеш`, якщо `cached > 0`), з розділювачами
    тисяч за локаллю.
  - **Вартість** — `$0.0123` — **тільки якщо `isAdmin` і `costUsd != null`**.
  - **Час роботи** — формат «12.3 с» / «1 хв 3 с» з `durationMs`.
  - **Надіслано** — повна дата+час (`Intl.DateTimeFormat`, повний), доповнює
    відносний час у `TimestampRow`.
- Поповер-примітив: UI-бібліотека проєкту — **Base UI** (`@base-ui/react`,
  існуючий `tooltip.tsx` імпортує з `@base-ui/react/tooltip`). У Base UI є
  `@base-ui/react/popover` — додати тонку обгортку `components/ui/popover.tsx`
  у тому ж стилі, що інші ui-примітиви (`cn`, варіанти класів). Жодних нових
  залежностей не потрібно.

### 4. Деградація

Старі повідомлення (до цієї зміни) не мають нових полів → (i) для них не
рендериться. Жодних міграцій БД не потрібно (усе в існуючому `jsonb metadata`).

## Поза обсягом (YAGNI)

- Жодних змін на повідомленнях користувача (без (i), без точного часу).
- Жодного бекфілу історичних токенів/часу в старі повідомлення.
- Жодних змін у сторінці аналітики `/settings/usage`.
- Поповер не показує розбивку по кроках/інструментах — лише підсумок турну.
